Thiết kế RESTful API với Spring Boot

Ngày đăng: 27/10/2025 Tác giả: Nguyễn Thiện Trí

Cấu trúc dự án

Tách lớp controller, service, repository; dùng DTO cho payload và entity cho persistence. Áp dụng validation với javax.validation.

// Controller mẫu
@RestController
@RequestMapping("/api/v1/users")
public class UserController {
  private final UserService service;
  public UserController(UserService service) { this.service = service; }

  @GetMapping
  public List<UserDto> list() { return service.list(); }

  @PostMapping
  public ResponseEntity<UserDto> create(@Valid @RequestBody CreateUserDto dto) {
    return ResponseEntity.status(HttpStatus.CREATED).body(service.create(dto));
  }
}

Error handling & tiêu chuẩn HTTP

Sử dụng @ControllerAdvice để gom lỗi, trả về mã trạng thái phù hợp (400, 404, 409, 500). Bao gồm traceId để theo dõi.

@ControllerAdvice
public class ApiExceptionHandler {
  @ExceptionHandler(MethodArgumentNotValidException.class)
  ResponseEntity<ApiError> handleValidation(...) { /* ... */ }
}

Rate limit, caching, observability

  • Rate limit: Bucket4j hoặc gateway (Nginx/API Gateway).
  • Caching: @Cacheable với Redis cho dữ liệu đọc nhiều.
  • Observability: Micrometer + Prometheus + Grafana theo dõi latency p95/p99.

Bảo mật

Bật Spring Security, kiểm tra quyền ở backend, tránh để lộ thông tin qua lỗi. Sử dụng JWT hoặc OAuth2/OpenID Connect.

Kết luận

Thiết kế chuẩn giúp API dễ bảo trì, an toàn và quan sát tốt. Bắt đầu đơn giản, thêm dần các lớp bảo vệ và giám sát.
