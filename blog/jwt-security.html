Bảo mật API với JWT

Ngày đăng: 31/10/2025 Tác giả: Nguyễn Thiện Trí

Ký và xác thực JWT

Luôn ký JWT với HMAC hoặc RSA; xác thực chữ ký, issuer, audience và expiry trên backend.

Quản lý thời hạn và refresh token

Đặt expiry ngắn cho access token; lưu refresh token an toàn (HTTP-only cookie) và rotate mỗi lần sử dụng.

Phân quyền và audit

Kiểm tra quyền ở backend; ghi log hành động quan trọng để audit và phát hiện bất thường.

Mẹo phòng thủ

  • Không lưu JWT trong localStorage nếu có thể bị XSS.
  • Thêm jti để hỗ trợ revoke theo danh sách đen.
  • Giới hạn phạm vi (scope) của token theo nguyên tắc tối thiểu.

Kết luận

JWT mạnh nhưng phải dùng đúng cách: ký, xác thực chặt chẽ và quản trị vòng đời token.
